# ToG Components

This table describes the major runtime stages in the ToG-2 pipeline. The primary entry point is `TOG_Original/ToG-2/main_tog2.py`, and the central per-question function is `main_wiki_new`.

| Stage Name | Purpose | Input | Output | Functions Used | Files Used | Runtime Variables | Failure Cases |
|---|---|---|---|---|---|---|---|
| Self Consistency | Ask the LLM multiple times before graph traversal and skip graph search when the answers agree strongly enough. | `query`, current dataset `data`, sample index `i`, CLI `args`. | `data_point` enriched with `cot_sc_score`, `cot_sc_response`, `cot_sc_answer`; possible early final answer. | `utils.self_consistency`, nested `get_s1_prompt`, nested `get_cot_sc_results`, nested `s1_reasoning_preparation`, `utils.run_llm`. | `ToG-2/main_tog2.py`, `ToG-2/utils.py`, `ToG-2/prompt_list.py`, dataset JSON files. | `args.self_consistency`, `args.self_consistency_threshold`, `args.max_length`, `args.opeani_api_keys`, `args.LLM_type`, `data_point["cot_sc_score"]`, `data_point["cot_sc_response"]`. | LLM API failure raises `"Stage 1: OpenAI API call failed"`; no extractable answers gives score `0`; low score continues to graph search; high score may return an incorrect but confident LLM answer. |
| Topic Pruning | Reduce noisy initial topic entities when a question has more than two linked entities. | `question`, `topic_entity` dictionary, CLI `args`. | Pruned topic entity dictionary/list, or original entities if pruning fails. | `wiki_func.topic_e_prune`, nested `construct_topic_prune_prompt`, nested `extract_output`, `wiki_func.run_llm_json`. | `ToG-2/main_tog2.py`, `ToG-2/wiki_func.py`, `ToG-2/prompt_list.py`. | `args.topic_prune`, `args.temperature_exploration`, `args.max_length`, `args.opeani_api_keys`, `args.LLM_type`, `topic_entity`. | JSON parse failure returns original entities; pruning may remove every entity, causing `generate_only_with_gpt`; LLM may select irrelevant entities. |
| Wikipedia Retrieval | Fetch textual evidence for topic entities and expanded candidate entities. | `wiki_client`, entity record such as `{"name": entity_name, "id": entity_id}`. | Raw Wikipedia text string; optionally cached text file under `docs_pages/`. | `wiki_func.get_wikipedia_page`, `client.MultiServerWikidataQueryClient.query_all`, `client.WikidataQueryClient.get_wikipedia_page`, `client.format_entity_name_for_wikipedia`. | `ToG-2/main_tog2.py`, `ToG-2/wiki_func.py`, `ToG-2/client.py`, `ToG-2/server_urls.txt`, generated `docs_pages/`. | `wiki_client`, `entity_id`, `entity_name`, `entity_dict`, `related_passage`, `docs_folder`. | Wikidata server unavailable; Wikipedia HTTP timeout/error returns `"Not Found!"`; missing page title; cache write failure is printed and ignored; empty or noisy page text reduces downstream ranking quality. |
| Embedding Search | Split retrieved Wikipedia text and rank paragraphs/sentences by relevance to the question. | `question`, `related_passage`, CLI `args`, `emb_model`, optional `top_k`. | Best paragraph text and top ranked sentence records, or paragraph lists for candidate ranking. | `search.pages_embedding_search`, `search.pages_embedding_search_only_para`, `search.split_paragraphs`, `search.split_sentences_windows`, `search.s2p_relevance_scores`, `search.scores_rank`, `search.biencoder_similarity`, `search.crossencoder_similarity`, `utils.compute_bm25_similarity`. | `ToG-2/main_tog2.py`, `ToG-2/search.py`, `ToG-2/utils.py`. | `args.embedding_model_name`, `args.sliding_window`, `emb_model`, `Total_Related_Senteces`, `sorted_sentences`, `paragraph`, `top_k`. | `related_passage == "Not Found!"` returns empty results; no valid paragraphs returns `('', [])`; unsupported embedding model raises `"Unknown embedding model"`; model loading/scoring errors stop execution. |
| Relation Pruning | Select the most promising Wikidata relations before expanding graph neighbors. | Current `topic_entity`, previous relation/head state, `question`, CLI `args`, `wiki_client`. | `current_entity_relations_list` containing relation records with entity id/name, relation label, score, and direction. | `wiki_func.relation_search`, `wiki_func.relation_search_prune`, `wiki_func.relation_prune_all`, `wiki_func.construct_relation_prune_prompt`, `wiki_func.construct_all_relation_prune_prompt`, `wiki_func.clean_relations`, `wiki_func.clean_relation_all_e`, `wiki_func.abandon_rels`, `utils.run_llm`. | `ToG-2/main_tog2.py`, `ToG-2/wiki_func.py`, `ToG-2/utils.py`, `ToG-2/prompt_list.py`, `ToG-2/client.py`. | `args.relation_prune`, `args.relation_prune_combination`, `args.remove_unnecessary_rel`, `args.width`, `pre_relations`, `pre_heads`, `all_entity_relations`, `current_entity_relations_list`. | No relations at depth 1 triggers `generate_only_with_gpt`; LLM output may not match expected relation-score format; low scores can filter all relations; Wikidata `get_all_relations_of_an_entity` may return `"Not Found!"`. |
| Entity Expansion | Traverse selected relations to collect neighboring Wikidata entities or literal finish values. | Relation records from relation pruning/search, `wiki_client`. | Candidate entity list and `Indepth_total_candidates` with relation/path metadata and retrieved paragraphs. | `wiki_func.entity_search`, `wiki_func.update_history_find_entity`, `wiki_func.get_wikipedia_page`, `search.pages_embedding_search_only_para`, `client.MultiServerWikidataQueryClient.query_all`. | `ToG-2/main_tog2.py`, `ToG-2/wiki_func.py`, `ToG-2/search.py`, `ToG-2/client.py`. | `relation["entity_id"]`, `relation["relation"]`, `relation["head"]`, `entity_candidates`, `each_relation_right_entityList`, `Indepth_total_candidates`. | Relation label cannot map to PID; no tail/head entities found; literal values produce `[FINISH_ID]`; candidates with no Wikipedia paragraphs are dropped; zero candidates triggers LLM-only fallback. |
| Candidate Ranking | Rank expanded candidate entities using their retrieved paragraphs and choose the next graph frontier. | `question`, `Indepth_total_candidates`, CLI `args`, `emb_model`. | `flag`, `cluster_chain_of_entities`, `entities_id`, `pre_relations`, `pre_heads`, `sorted_entity_list`, updated candidates. | `wiki_func.para_rank_topk`, `wiki_func.para_rank_topk_fin`, `search.s2p_relevance_scores`, `search.split_sentences_windows`, `search.split_sentences`. | `ToG-2/main_tog2.py`, `ToG-2/wiki_func.py`, `ToG-2/search.py`. | `args.width`, `args.sliding_window`, `args.embedding_model_name`, `entities_id`, `relations`, `heads`, `sorted_entity_list`, `cluster_chain_of_entities`. | Empty ranked entity list returns `flag=False`; missing paragraphs produce weak/empty scores; all selected IDs may be `[FINISH_ID]`; ranker errors stop execution. |
| Reasoning | Ask the LLM whether the collected graph paths and evidence are sufficient to answer the question. | `original_question`, `Indepth_total_candidates`, `Total_Related_Senteces`, `cluster_chain_of_entities`, CLI `args`, `clue`. | `(stop, answer, kg_prompt)` where `stop=True` ends graph traversal. | `wiki_func.reasoning`, `wiki_func.reasoning_fin`, nested `num_tokens_from_string`, `utils.run_llm`, `utils.run_llm_cnfin`, `utils.extract_answer`, `utils.if_true`, `wiki_func.contains_yes_regex`. | `ToG-2/main_tog2.py`, `ToG-2/wiki_func.py`, `ToG-2/utils.py`, `ToG-2/prompt_list.py`. | `args.dataset`, `args.clue_query`, `args.num_sents_for_reasoning`, `args.temperature_reasoning`, `args.max_length`, `chain_prompt`, `related_sentences_prompt`, `final_prompt`, `response`. | LLM may not use expected brace/yes format, causing `stop=False`; token budget can omit useful sentences; prompt condition bug treats `('fever' or 'creak') in args.dataset` as only checking `"fever"`; API failure returns empty response. |
| Answer Generation | Produce final answer either after successful reasoning or via fallback paths when graph traversal cannot continue. | `question`, optional evidence/path records, `ground_truth`, `end_mode`, `remark`, CLI `args`. | Final `answer` and saved JSON output record. | `utils.generate_only_with_gpt`, `utils.generate_without_explored_paths`, `utils.generate_only_with_sentences`, `search.save_2_jsonl_simplier`, `utils.save_2_jsonl`. | `ToG-2/main_tog2.py`, `ToG-2/utils.py`, `ToG-2/search.py`, `ToG-2/prompt_list.py`, output JSON file. | `answer`, `end_mode`, `remark`, `search_entity_list`, `Total_Related_Senteces`, `cluster_chain_of_entities`, `args.output`, generated filename fields. | Fallback answer may ignore retrieved graph evidence; LLM API failure can return empty string; output append logic rewrites JSON arrays and can preserve malformed content if the existing file is malformed; FEVER fallback may produce `NOT ENOUGH INFO` when evidence is insufficient. |
